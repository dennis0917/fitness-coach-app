(우리 앱 = 한국어 단일 JSX/React 모바일 우선 웹앱 / 타겟: 188cm·113→83kg·운동초보·무릎보호 / 주4회 4분할·유산소·식단(계란제외)·휴식타이머·점진과부하·운동 시각자료·체중추적)

## A. 정보구조(IA)·네비게이션 패턴 — 검증된 원칙

- **하단 탭바(Bottom Navigation)가 표준**이며, 탭은 **3~5개 이내**로 유지. "탭바에 버튼이 3개를 넘으면 복잡하다"는 가이드가 반복 등장. 대표 구성: Home / Workouts / Progress(Gains) / Settings.
- **확장 가능한 IA 설계**: 운동·진행추적·식단·커뮤니티가 늘어나도 "사용자가 길을 잃지 않도록" 탭+허브+바로가기+가이드 플로우를 하나의 논리 체계로 구성하라는 권고.
- **적응형 네비게이션(Adaptive nav)**: 사용자 행동에 따라 가장 관련 높은 옵션을 먼저 노출.
- **우리 앱 적용**: 단일 화면 React 웹앱이므로 하단 고정 탭 **4개**가 적정 → `홈 / 운동 / 식단 / 진행`. 휴식타이머·다음무게 제안은 별도 탭이 아니라 **운동 세션 화면 안의 컨텍스트 UI**로 넣어 탭 수를 늘리지 않는다. 운동 시각자료(자세/근육)는 운동 항목을 탭하면 열리는 상세/모달로.

## B. 홈 대시보드 베스트프랙티스 — "한눈에 보이는 홈"

- 홈은 **선택과 집중**: 활동 추적형은 개인 통계, 운동 중심형은 "오늘의 운동(Workout of the day)/추천"을 최상단에.
- 결과 대시보드는 **"남이 아닌 과거의 나와 비교"** — 개인 성장 중심으로 목표 달성·통계·성취를 명확히.
- **데일리 리텐션 장치**: 대시보드 자체가 매일 돌아오게 만드는 핵심. 스트릭·배지·챌린지로 일관성 보상.
- **우리 앱 적용(홈 위 → 아래 순서 제안)**: ①오늘의 분할(예: "오늘=가슴/삼두") + 시작 버튼 → ②체중 트렌드 미니그래프(목표 83kg까지 진척률) → ③오늘 칼로리/매크로 링 → ④이번 주 운동 4회 중 완료 진행바 + 스트릭 → ⑤"지난번보다 +2.5kg 들었어요" 식 과거-나 비교 한 줄. 초보+고체중 동기부여이므로 **비교 대상은 항상 본인**으로 못박기.

## C. 다크모드·비주얼 트렌드 (2025~2026)

- **다크모드 + 그린/블루 액센트**가 헬스장 환경(저조도)에서 가독성·눈 피로 측면 표준으로 언급.
- iOS 26의 **Liquid Glass(반투명/유리질)** 스타일로 위젯 재디자인 흐름(서드파티 대시보드 앱 Peak 등). [추정: 우리 웹앱에 직접 도입은 선택, 단 과한 블러는 저사양/가독성 주의]
- **우리 앱 적용**: 다크 기본 + 단일 강조색(예: 그린=긍정/달성). 큰 숫자 타이포(현재 체중, 오늘 칼로리)로 "한눈" 강화. 운동 중 화면은 멀리서도 보이게 대비/폰트 크게.

## D. 데이터 시각화 — 링·도넛·트렌드·진행바·스트릭

- **활동 링(Apple)**: Move/Exercise/Stand 3중 링이 "닫는다"는 명확한 멘탈모델. 위젯/잠금화면까지 확장.
- **매크로 = 원형(도넛) 차트**: 칼로리·단백질·탄수를 굵은 색으로 도넛 분할해 "묻히지 않게" 표현이 트렌드.
- **체중 = 트렌드(스무딩) 그래프**: Happy Scale/Libra가 **지수평활·7일 이동평균** 등으로 일일 변동을 걸러 "진짜 추세"를 보여줌. 고체중 감량자는 일일 수분변동에 흔들리므로 필수.
- **진행/성취**: 차트·배지·스트릭이 습관 형성 동력.
- **우리 앱 적용**: ①식단=매크로 도넛+칼로리 잔여, ②체중=**입력값 점 + 이동평균 추세선**(단순 raw 라인 금지) + 목표선(83kg) + 예상 도달일, ③근력=세트 볼륨/1RM 추정 라인차트, ④주간 4회 진행바 + 스트릭. (Apple 링 컨셉을 "오늘 할 일 3개=근력/유산소/식단" 닫기 링으로 변형 가능 [추정])

## E. 접근성 (WCAG 2.2)

- **터치 타깃**: WCAG 2.5.8(AA) 최소 24×24 CSS px, 권장은 **Apple 44×44pt / Material 48×48dp**. 보이지 않는 패딩으로 히트영역 확보.
- **고대비·큰 타이포·명확한 폰트**, 음성 가이드, 스크린리더 호환 권장.
- **이미지/영상 alt 텍스트 필수**(한 앱은 운동 이미지/영상 5만+ alt 누락으로 출시 4개월 지연 사례).
- **우리 앱 적용**: 운동 중 버튼(세트 완료/+무게/타이머)은 **48px 이상**, 장갑/땀 손가락 고려해 간격 충분히. 운동 시각자료엔 한국어 alt("벤치프레스 - 가슴/삼두 자극") 부여.

## F. 앱별 분석

### 1) Hevy — 근력 로깅 + 소셜 (iOS/Android, 웹)
- **핵심기능**: 빠른 세트 로깅, **이전 수행값 자동 채움**(점진과부하 핵심), 자동 휴식타이머, 진행 차트, 1RM 추정, 루틴 템플릿, 소셜 피드. 무료 티어가 강력.
- **우리 앱 적용**: "지난번 무게/횟수 자동 표시 → 그 위에 다음 제안값 오버레이"가 **점진과부하 UX의 정석**. 우리도 직전값을 기본 placeholder로 깔고 ⑤ 자동 제안을 적용.
- **강점**: 무료 범위 넓음, 깔끔·빠른 로깅, 커뮤니티 책임감.
- **약점/주의**: **프로그램을 만들어주진 않음**(스스로 계획 가정) → 초보엔 부담. 우리 앱은 "가르치기"가 핵심이라 이 갭을 메워야 함.
- **수익모델**: 프리미엄 $23.99/년, 평생 $74.99(트레이너 포함).

### 2) Strong — 순수 로깅 UX (iOS/Android/Watch)
- **핵심기능**: 가장 정제된 로깅 흐름, **휴식타이머·원판(plate) 계산기·세트 추적·이전 기록 보기**가 매끄러움.
- **우리 앱 적용**: **원판 계산기**(바벨에 어떤 원판 끼울지)와 자동 시작되는 휴식타이머는 초보에게 특히 유용 → 채택 권장.
- **강점**: 인터페이스 폴리시 최상, 빠름.
- **약점/주의**: **주기화/프로그래밍/성과 기반 강도조절 없음** = 빈 캔버스 → 중급+ 대상. 초보 단독 사용 부적합.
- **수익모델**: $29.99/년 + 평생 옵션.

### 3) Fitbod — AI 자동 프로그래밍 (iOS/Android)
- **핵심기능**: 이력·**근육 피로/회복 추정**·목표로 매 세션 운동 자동 생성, 점진과부하 자동 관리, **근육 회복 히트맵(머슬맵)**, 1,600+ 종목 HD 영상, 장비 적응(풀짐~맨몸), 볼륨/PR 분석.
- **우리 앱 적용**: ①**근육 회복 히트맵**(어디가 회복/피로)은 4분할 루틴과 찰떡 → "오늘 어디 운동" 시각 근거 제공. ②"AI가 다음 운동을 정해준다"는 초보 친화 모델 = 우리의 ⑤ 자동 제안과 부합. ③장비 적응 로직은 홈/헬스장 양쪽 대응에 참고.
- **강점**: 핸즈오프(생각 없이 따라하면 됨), 회복 기반 분할 자동화.
- **약점/주의**: **가격 비쌈**, AI가 통제권을 가져가 사용자 학습/주도성 약화 가능. 우리는 "가르치기"로 통제권을 점진 이양하는 게 차별점.
- **수익모델**: $15.99/월 또는 $95.99/년, 7일 무료체험.

### 4) JEFIT — 운동 라이브러리 + 바디맵 (iOS/Android/웹)
- **핵심기능**: **1,500+ 종목 애니메이션 시연 + 근육 활성 다이어그램**, **바디맵(전/후면 해부도에서 근육 탭 → 해당 종목 표시)**, 초/중/고급 플랜, **운동 중 노란 다이아몬드 아이콘으로 종목 즉시 교체(스왑)**, 워크아웃 빌더.
- **우리 앱 적용**: ⑥운동별 시각자료의 모범 — **종목 상세에 (a)동작 애니메이션 (b)자극 근육 하이라이트 그림**을 함께. **무릎보호 핵심**: "고임팩트 → 무릎 친화 대체 종목 스왑" UX를 그대로 차용(예: 바벨스쿼트→레그프레스/고블릿/박스스쿼트 제안).
- **강점**: 시각 자료·해부도 풍부, 초보~고급 플랜, 종목 교체 유연.
- **약점/주의**: 기능 많아 UI가 무겁고 복잡하다는 평 [추정]. 우리는 4분할에 맞춰 큐레이션 필요.
- **수익모델**: 프리미엄 구독(프리미엄 Elite). [추정: 정확 단가 미확인]

### 5) MacroFactor — 적응형 식단 코치 (iOS/Android)
- **핵심기능**: **실제 체중 데이터로 TDEE를 동적 추정→매주 목표 칼로리/매크로 자동 조정**(코치형), 시장 최速 로깅(검색 평균 10액션, AI 음성/서술 로깅), 54개 영양소 추적, 자체 체중 트렌드.
- **우리 앱 적용**: "단순 기록(ledger)"이 아니라 **"체중 추세를 보고 다음 주 칼로리를 조정해주는 코치"** 컨셉이 ③식단+⑦추적+⑤제안을 하나로 묶는 핵심 인사이트. **계란 제외** 같은 제약은 음식 추천 필터로 구현(MacroFactor엔 없는 우리만의 개인화).
- **강점**: 빠른 로깅, 적응형 알고리즘, 정확도 높은 DB.
- **약점/주의**: **북미 외 음식 DB 빈약**(유럽/국제 사용자 "수동입력 없으면 사실상 불가") → **한국 음식 DB는 우리가 직접 구축/큐레이션 필요**(가장 큰 현지화 리스크).
- **수익모델**: $11.99/월, $71.99/년, 7일 무료체험.

### 6) MyFitnessPal — 식단 기록 베이스라인 (iOS/Android/웹)
- **핵심기능**: **2,000만+ 식품 DB**(최대 규모), 바코드, 기본 매크로(14항목).
- **우리 앱 적용**: DB 규모가 곧 로깅 마찰 감소임을 보여줌 → 한국 식품 검색/바코드의 **커버리지**가 우리 식단 기능의 성패. 단, MFP의 약점(아래)을 반면교사로.
- **강점**: 압도적 DB, 무료 기본 기능.
- **약점/주의**: 사용자 입력 DB라 **부정확 항목 많음**, 14항목만 추적(비타민/미네랄 제외), 광고, "코치가 아닌 장부".
- **수익모델**: 프리미엄(광고제거/전기능).

### 7) Apple Fitness(활동 링/위젯) — 대시보드·시각화 레퍼런스 (iOS/watchOS)
- **핵심기능**: **Move/Exercise/Stand 3중 링**, 홈/잠금화면 위젯(링 위젯 + 수치 위젯), 활동 요약. (서드파티 Peak/Hidgets가 위젯 대시보드 확장, iOS 26 Liquid Glass)
- **우리 앱 적용**: **"링을 닫는다"는 단순·강력한 멘탈모델**을 우리 일일 목표(근력/유산소/식단 3링)로 변형. 큰 위젯=수치+목표 근접도, 작은 위젯=링만 — 정보 밀도 2단계 구성 참고.
- **강점**: 직관적, 중독성 있는 완료 동기.
- **약점/주의**: 워치/하드웨어 종속, 링이 "활동량" 중심이라 근력 진척 표현엔 부적합 → 우리는 **볼륨/무게 진척**을 별도 시각화로 보완해야.
- **수익모델**: iOS 기본 무료(코칭 영상 Fitness+는 별도 구독).

### 8) Nike Training Club — 비디오 코칭/가르치기 (iOS/Android, 무료)
- **핵심기능**: 마스터 트레이너의 **비디오 코칭**, **선제적 폼/타이밍 큐**("다음 동작 예고"), 스킬레벨 필터, **운동 중 난이도 수정(modification)**, 수백 개 영상, 2026년 전 라이브러리 결제정보 없이 무료.
- **우리 앱 적용**: ①가르치기의 정석 — **"화면 못 볼 때를 대비한 선제적 큐"**(텍스트/음성으로 다음 세트·자세 포인트 미리 안내). 초보 포용적 modification = 무릎보호용 강도 하향 옵션과 연결.
- **강점**: 코칭 품질·제작 퀄리티 높음, 무료, 초보 포용적이되 시혜적이지 않은 톤.
- **약점/주의**: 영상 중심이라 **로깅/점진과부하 추적은 약함** → 우리는 코칭+로깅을 결합해야(NTC와 Hevy의 장점 합치기).
- **수익모델**: 완전 무료(Nike 브랜드/커머스 연계).

### 9) Happy Scale / Libra — 체중 트렌드 시각화 (iOS / Android)
- **핵심기능**: **스무딩(지수평활·7일 이동평균·이중지수 등)으로 일일 변동 제거**, 추세선·예측(목표 도달일), 표시 기간 조절(원데이터는 불변).
- **우리 앱 적용**: ⑦체중추적의 핵심 — **raw 값이 아니라 추세선**을 메인으로, "오늘 +0.4kg"이 아니라 "추세 -3.2kg/지난 30일"을 보여줘 고체중 감량자의 좌절 방지. 목표 83kg 선 + 현재 페이스 기반 **예상 도달일** 표기.
- **강점**: 변동에 흔들리지 않는 심리적 안정 + 동기.
- **약점/주의**: 체중 단일 기능(우리는 종합 앱이므로 미니그래프로 임베드).
- **수익모델**: Happy Scale 프리미엄(일부 무료) / Libra 무료~프리미엄. [추정]

## 이 카테고리 핵심 교훈 3가지

1. **"한눈에 보이는 홈" = 오늘 할 일 + 과거의 나와 비교 + 추세**. 하단 탭 4개(홈/운동/식단/진행)로 단순화하고, 홈 최상단엔 "오늘의 분할+시작", 그 아래 체중 추세·매크로 링·주간 진행바를 배치. 비교 기준은 **언제나 본인의 과거**(고체중 초보 동기부여의 핵심).

2. **점진과부하·식단·체중은 "기록(ledger)"이 아니라 "코치(coach)"로 묶어라**. Hevy(이전값 자동채움)+Fitbod(회복기반 다음 운동/무게 제안)+MacroFactor(체중추세로 칼로리 자동조정)의 공통 패턴을 결합 → "지난번 값 → 다음 제안 → 수행 → 추세 반영"의 폐루프. NTC식 **선제적 코칭 큐**로 "가르치기"를 입힌다.

3. **무릎보호·초보·시각화는 곧 안전과 신뢰**. JEFIT처럼 **동작 애니메이션+자극 근육 해부도**를 종목마다 제공하고, **고임팩트→무릎친화 대체 종목 스왑**을 1탭으로. 다크모드+큰 타이포+48px 터치타깃+한국어 alt로 운동 중 가독성과 접근성을 확보. (현지화 최대 리스크 = **한국 음식 DB 자체 구축**, MacroFactor의 비북미 DB 공백이 경고 사례.)
