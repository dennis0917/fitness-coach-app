## 3. 차별화 포인트 (이 사용자 특화)

1. **고체중 무릎 보호를 1급 기능으로** — 경쟁 앱 전체의 공통 약점("데모는 있으나 폼/안전 코칭이 얕다")을 정조준. 운동마다 `kneeRisk` 플래그 + **무릎친화 대체운동 1탭 스왑** + 워밍업 세트 자동생성 + 저충격 유산소 1순위. 온보딩 첫 화면부터 무릎 질문 노출 → 신뢰 형성.
2. **4분할 고정으로 선택과부하 제거** — Jefit식 "기능 많아 길 잃음"의 반면교사. 가슴삼두/등이두/하체/팔어깨 4개만 큐레이션, "오늘 뭐 하지"를 고민할 필요 없음. 초보에겐 선택지를 좁히는 게 곧 친절.
3. **계란 제외 한식 + 단백질 우선 + 결핍 안전망** — 계란 알레르겐 자동 필터 + 단백질 최상단 게이지 + 계란 제외로 생길 비타민D/B12·콜린 달성률 모니터. 한식 인식은 글로벌 AI보다 한국 특화 엔진.
4. **"가르치고 자동화하는" 코치** — 단순 로거(번핏/Strong)도, 블랙박스 AI(Fitbod)도 아님. **규칙형 자동제안(왜 +2.5kg인지 설명) + 1분 레슨/퀴즈 + 근육맵**으로 통제권을 점진 이양. 초보가 따라 하며 배운다.
5. **30kg 장기여정 동기 설계** — 멀리 있는 83kg를 매일의 성취로 쪼갬: 유연 스트릭(주4회) + 스트릭 프리즈 + 85% 부분성공 인정 + 추세체중(일일 변동에 안 흔들림) + 정체기 자동 코칭(3~6개월차 이탈 방어) + 체성분으로 "보이지 않는 진전" 증명.

## 4. 정보구조(IA) — 하단 탭 4개 + 컨텍스트 UI

탭은 3~5개 표준 → **홈 / 운동 / 식단 / 진행** 4개. 휴식타이머·다음무게 제안은 별도 탭이 아니라 **운동 세션 화면 내 컨텍스트 UI**, 시각자료는 운동 탭 시 모달.

```
┌ 하단 탭 ────────────────────────────────────┐
│  🏠 홈    💪 운동    🍱 식단    📈 진행      │
└──────────────────────────────────────────────┘
```

**🏠 홈 (대시보드)** — 위→아래: ①오늘의 분할 + [운동 시작] ②체중 추세 미니그래프(83kg 진척률) ③오늘 칼로리/매크로 도넛 링(단백질 강조) ④주간 4회 진행바 + 스트릭 ⑤"지난번보다 +2.5kg" 과거-나 비교 ⑥(확장) 1분 레슨 카드/정체기 코칭 배너

**💪 운동**
- 4분할 카드(가슴삼두/등이두/하체/팔어깨), 오늘 분할 강조
- → 분할 진입: 종목 리스트(세트수·목표횟수·휴식 프리셋)
- → 종목 상세(모달): **SVG 근육맵 + 자세 2컷 + 한국어 단계 큐 + 무릎 경고/대체운동 스왑 + 유튜브 영상 버튼**
- → **세션 실행 화면**(컨텍스트 UI 집약): 직전기록 자동채움 → 세트 완료 체크 → **휴식 타이머 자동시작(대형 카운트다운)** → 다음무게 제안 + RiR("몇 개 더?") → 플레이트 계산기 → 워밍업 세트
- (확장) 유산소 서브탭: 저충격 종목 + 걸음/거리 목표

**🍱 식단**
- 단일 로깅 진입점(검색 우선, 확장 시 바코드/사진/음성)
- 끼니별 기록 + **계란 필터** + 신호등 색 + 칼로리 잔여
- 매크로 도넛(단백질 최상단) + (확장) 미량영양소 달성률 바

**📈 진행**
- 체중: 입력 점 + 7일 이동평균 추세선 + 목표선 + 예상도달일
- 근력: 종목별 볼륨/추정1RM 라인차트, PR
- (확장) 체성분/둘레/사진 타임라인 + 월1 리포트 카드
- 뱃지/마일스톤/스트릭 현황

**접근성:** 세션 화면 버튼(세트완료/±무게/타이머)은 **48px 이상**, 땀/장갑 고려 간격 충분. 다크모드 기본 + 단일 강조색(그린=달성), 큰 타이포. 운동 이미지에 한국어 alt("벤치프레스 - 가슴/삼두 자극").

## 5. 데이터 모델 (localStorage 키 개요)

> 단일 JSX/React + localStorage. 운동 카탈로그는 **앱 번들 정적 JSON**(free-exercise-db 가공), 사용자 데이터만 localStorage. 사진/바이너리는 **IndexedDB** 권장(localStorage 5~10MB 한계).

```
app:settings           { theme:"dark", units:"kg", soundOn, vibrationOn, notifOptIn }
user:profile           { sex:"M", height:188, startWeight:113, goalWeight:83,
                         birthYear, kneeIssue:true, experience:"beginner",
                         daysPerWeek:4, excludeFoods:["egg"], motivation, createdAt }
user:onboarding        { completed:true, estMonths:9, initialPlanId, tdeeInitial }

exercise:catalog       (정적 번들·읽기전용) [{ id, nameKo, category, primaryMuscles[],
                         secondaryMuscles[], equipment, instructionsKo[],
                         kneeRisk:0|1|2, alternatives:[exId], images[], ytQuery }]

routine:splits         [{ id:"chest_tri"|"back_bi"|"legs"|"arms_sh",
                         nameKo, exercises:[{ exId, sets, repTarget:[8,12],
                         restSec, defaultWeight }] }]

workout:sessions       [{ id, date, splitId, status,
                         entries:[{ exId, sets:[{ weight, reps, rir, done, isWarmup }] }],
                         totalVolume, durationSec }]
workout:lastByExercise { [exId]: { weight, reps, rir, date } }   // 직전기록 자동채움용
progression:state      { [exId]: { suggestedWeight, suggestedReps, rule, est1RM } }  // 규칙형 제안

diet:logByDate         { "2026-06-28": { meals:[{ foodId, name, grams, kcal,
                         protein, carb, fat, trafficLight:"green"|"yellow"|"red",
                         containsEgg:false }], totals, target } }
diet:customFoods       [{ id, name, per100g:{kcal,p,c,f}, tags:["egg-free"] }]
diet:targets           { kcal, protein, carb, fat, updatedAt }   // 적응형 칼로리가 갱신

weight:log             [{ date, kg }]                            // 추세선은 7일 이동평균 계산
weight:trend           { trendKg, paceKgPerWk, etaDate }         // 파생·캐시
bodycomp:log           [{ date, bodyFatPct?, muscleKg?, waistCm?, photoRef? }]

tdee:state             { estimatedTDEE, lastRecalcDate, history:[{wk, tdee, intake, trendDelta}] }
coaching:state         { plateauActive, lastLessonId, lessonsRead:[], quizScores:{} }

gamify:streak          { workoutStreak, dietStreak, freezesLeft, lastWorkoutDate,
                         scheduleDays:[1,2,4,5] }
gamify:badges          [{ id, earnedAt }]
gamify:milestones      { weightMilestonesHit:[110,105,...], totalVolumeKg }

meta:export            { lastExportAt, schemaVersion }           // JSON/CSV 내보내기
```

핵심 파생 루프: `workout:sessions` → `workout:lastByExercise` → `progression:state`(규칙형 제안) / `weight:log` + `diet:logByDate` → `tdee:state` → `diet:targets`(적응형 갱신).

## 6. 단계별 구현 로드맵 (MVP → 확장)

**Phase 0 — 토대 (1~2주)**
- 데이터 스키마 + localStorage 래퍼(버전·마이그레이션·**JSON 내보내기**)
- 온보딩 플로우(목표→무릎체크→일일칼로리 처방 카드)
- 4탭 셸 + 다크모드 디자인 시스템(48px 타깃, 큰 타이포)

**Phase 1 — MVP 코어 "혼자 따라 하기" (3~5주)**
- 4분할 고정 루틴 + 종목 상세(SVG 근육맵 + PD 이미지 + 한국어 큐 + 무릎경고/대체스왑 + 유튜브 딥링크)
- 세션 실행: 직전기록 자동채움 → 세트 로깅 → **휴식타이머(화면 카운트다운+소리/진동)** → **규칙형 다음무게 제안** → 플레이트 계산기 → 워밍업 자동생성
- 식단 기본 로깅(검색+계란필터+신호등+단백질 게이지, 정적/소형 한식 시드 DB)
- 체중 추적(추세선+목표선+예상도달일)
- 홈 대시보드 + 유연 스트릭 + 체중 마일스톤 뱃지

**Phase 2 — 코치화 "지속과 정체기 돌파" (3~4주)**
- **적응형 칼로리**(주간 TDEE 역산→목표 자동갱신) + 정체기 감지 코칭 메시지
- 마이크로러닝 1분 레슨+퀴즈, 능동 코치 톤, 근접완료 알림(PWA/Notification 범위 내)
- 음성 코칭(세트/휴식 읽어주기), 스트릭 프리즈 + 85% 부분성공 로직
- 미량영양소 안전망 게이지, 진행 탭 라인차트(볼륨/1RM)

**Phase 3 — 확장 "종합 코치" (4주+)**
- 저충격 유산소 모듈(자전거/걷기 + 리워드), 체성분/둘레/사진 + 월1 리포트
- 사진→칼로리 인식(한식 엔진 연동) + 분량 보정, 바코드

**Phase 4 — 고도화/백엔드 (백엔드 도입 시점)**
- 한식 DB 백엔드(MFDS 공공API + FatSecret 하이브리드 + 교차검증)
- 카메라 자세분석, 폼 영상 셀프비교, 소셜 챌린지, RPG 게이미피케이션(선택)

## 7. 리스크 / 주의

**저작권 (시각자료)**
- 운동 GIF/영상을 **다운로드·재호스팅 금지**(침해 위험 급증). ExerciseDB GIF는 "앱 내 표시 가능, 재배포 금지"라 **오픈 포털엔 부적합**.
- 안전 스택: **free-exercise-db(Unlicense/퍼블릭도메인) 정지이미지 + react-body-highlighter(오픈소스 SVG) + YouTube 공식 iframe 임베드 또는 검색 딥링크**. (※ 이미지 개별 라이선스 명시가 불완전하니 출처 표기·교체 가능 구조로)
- YouTube: 임베드는 일반적으로 링크로 취급되나 **SDNY 등 일부 관할은 임베드도 공개전시로 본 판례** 존재 → 비상업·딥링크 위주로 보수적 운용, 재호스팅 금지.

**식품 DB 정확도**
- 크라우드소싱 DB는 동일음식 **15~30% 편차**(하루 300~500kcal 오차). 사진 인식은 단일음식 ±10~15%, **복합·혼합요리 ±25~35%**, 분량 추정 특히 약함.
- 대응: **검증 출처 우선**(MFDS 공공DB 12,200 요리·무료 + FatSecret verified + 교차검증). 사진은 "1차 추정+분량 슬라이더 보정" 보조수단으로만, **단백질 정밀추적은 검색/직접입력이 신뢰축**. 한식은 글로벌 AI보다 FoodLens형 한국 특화 엔진.

**의료 면책 / 안전**
- 운동·식단 코칭은 의료 조언이 아님 → 온보딩·푸터에 **면책 고지**. **고체중(113kg) 무릎·심혈관 부하** 고려해 "운동 시작 전 의사 상담" 권고 배너, 통증 시 중단 안내.
- 무릎 위험 운동엔 강제 경고 + 대체운동. 장시간 단식 등 부적합 패턴은 권장하지 않음. GLP-1/의약품 영역은 다루지 않음.

**플랫폼/기술 한계**
- **웹앱 알림 한계**: 백그라운드·잠금화면 푸시 제한 → 휴식타이머는 **화면 내 대형 카운트다운 + 소리/진동**으로 보완, PWA + Notification API로 부분 대응.
- **localStorage 한계**: 용량(~5~10MB)·캐시 삭제 시 데이터 소실·기기간 동기화 불가 → **사진은 IndexedDB**, **JSON/CSV 내보내기**를 MVP에 포함, 스키마 버전·마이그레이션 설계.
- **적응형 칼로리 기대치**: 3~4주 일관 로깅 전제, 초기 체감 늦음 → 초기엔 공식 기반 처방 후 점진 전환, "왜 시간이 걸리는지" 안내.

**리텐션/행동 리스크**
- **올오어낫씽 이탈**(초보·고체중 치명): 스트릭 프리즈(이탈 21%↓ 근거), 85% 부분성공 인정, 부드러운 패널티+쉬운 복귀 — **처음부터** 내장.
- **과알림 피로** → 옵트인 + 빈도 조절. **정체기 포기**(3~6개월차 최대 이탈 구간) → 자동 감지 코칭으로 방어.
- **선택과부하**(Jefit 반례) → 4분할 고정으로 선택지 축소.
- 게이미피케이션은 코칭(본질)을 **대체하지 않고 병행** — 보상의 상업화(챌린저스 피벗) 경계.
