# 운동 시범/시각자료 표현 방식 조사 (2024~2026 최신 기준)

타겟(188cm·113kg→83kg, 운동 초보, 무릎 보호 필요)을 위한 "운동을 어떻게 가르칠 것인가" 관점으로, 주요 앱의 시각자료 방식과 단일 React 웹앱에서의 현실적 구현 전략을 정리했다.

---

## 1. Hevy — 운동 로그 + GIF 애니메이션

- **카테고리/플랫폼**: 근력 운동 트래커 / iOS·Android·웹
- **핵심기능**: 운동별 "How to" 탭에서 **데모 애니메이션(GIF류) + 단계별 텍스트 설명** 제공. 400+ 기본 운동(바벨/덤벨/머신/밴드/맨몸). 커스텀 운동에 사진·동영상·GIF 직접 첨부 가능.
- **우리 앱 적용점(takeaway)**: "데모 애니메이션 + 짧은 텍스트 큐" 조합이 초보 학습의 표준 패턴. 각 운동 카드에서 한 화면에 "동작 루프 + 핵심 큐 3~4줄"을 같이 보여주는 레이아웃을 그대로 차용할 것.
- **강점**: 운동 흐름을 끊지 않고 데모를 즉시 확인. 깔끔한 UX, 무료 범위가 넓음.
- **약점/주의**: 애니메이션 자체는 자체 제작 자산이라 우리가 그대로 못 씀(저작권). 초보용 "왜/주의점" 코칭 깊이는 얕은 편.
- **수익모델**: 프리미엄 구독(루틴 무제한·분석).

## 2. JEFIT — HD 비디오 + BodyMap

- **카테고리/플랫폼**: 근력 트래커/플래너 / iOS·Android·웹
- **핵심기능**: 1,400+ 운동, **2023년 도입한 HD 비디오 데모(여러 각도)** + 텍스트 지시. **BodyMap**으로 근육 탭 → 해당 근육에 기여하는 운동을 기여도(%)순으로 표시. 2025년 점진적 과부하 시스템 개선.
- **우리 앱 적용점**: ①"근육 → 운동" 역방향 탐색(BodyMap)은 우리의 ⑥시각자료·⑤점진적 과부하와 직결되는 좋은 모델. ②근육별 **기여도 %** 개념을 4분할 루틴 설명에 응용하면 초보가 "이 운동이 어디에 듣는지"를 직관적으로 이해.
- **강점**: 운동 DB 방대, 다각도 비디오, 무게/세트 로깅 강함.
- **약점/주의**: **무료 버전은 비디오 시청 불가**(Elite 구독 필요). 데이터 많아지면 성능 이슈 보고됨.
- **수익모델**: Elite 구독(비디오·분석·플랜).

## 3. Fitbod — AI 자동 루틴 + 비디오 데모 + 근육 피로 히트맵

- **카테고리/플랫폼**: AI 근력 코칭 / iOS·Android
- **핵심기능**: 1,000+ 운동, **HD 비디오 데모 + 텍스트 코칭 큐(템포·가동범위)**. 목표·장비·**회복 상태(근육 피로 히트맵)** 기반으로 매 운동을 AI가 자동 생성·조정. 어느 근육이 "신선/피로"인지 인체도에 색으로 표시.
- **우리 앱 적용점**: ①**근육 피로/회복 히트맵**은 주4회 4분할에서 "오늘 어디를 쉬게 하는지"를 시각화하는 강력한 도구 → SVG 인체도에 색 강도로 표현 가능. ②"데모는 빠르게 접근, 로깅 흐름 방해 안 함"이라는 UX 원칙.
- **강점**: 업계 최고 수준 UX(2026년에도 기준점으로 평가), 자동 점진적 과부하.
- **약점/주의**: 데모가 "기능적"일 뿐 **초보가 가장 필요로 하는 폼 코칭 깊이는 부족**하다는 평. 우리 앱은 이 빈틈(무릎 보호·초보 큐)을 메우는 게 차별점.
- **수익모델**: 구독(약 $16/월 수준).

## 4. MuscleWiki — "근육 클릭 → 운동" 인터랙티브 인체도의 원형

- **카테고리/플랫폼**: 무료 운동 라이브러리 / 웹 + iOS·Android
- **핵심기능**: **인터랙티브 바디맵에서 근육 클릭 → 해당 부위 운동 + GIF/동영상 데모 + 단계별 폼 지시**. 1,600~2,000+ 운동 영상. 칼로리·매크로·1RM 계산기, 앱엔 AI 트레이너. 웹은 **가입 없이 100% 무료**.
- **우리 앱 적용점**: 우리가 추구하는 **"⑥운동별 시각자료 + 근육 타겟 인체도"의 가장 직접적인 벤치마크.** 앞/뒤 인체도에서 부위 탭 → 운동 리스트 → 데모 패턴을 핵심 네비게이션으로 채택할 것. 무료/무가입 접근성도 우리 비상업 포털 성격과 부합.
- **강점**: 진입장벽 0, 근육-운동 매핑이 직관적, GIF가 짧고 명확.
- **약점/주의**: GIF 자산은 자체 촬영물이라 **우리가 복제 사용 불가**. 초보 안전·무릎 보호 같은 맥락 코칭은 약함.
- **수익모델**: 앱 프리미엄/광고 [추정].

## 5. Muscle & Motion (Strength) — 3D 해부학·생체역학의 최상위

- **카테고리/플랫폼**: 해부학 학습형 근력 앱 / iOS·Android·웹
- **핵심기능**: 1,200+ 운동을 **3D 애니메이션**으로 분해 — 주동근/협력근 활성, **흔한 실수, 대체 운동**까지. 회전·줌 가능한 advanced 3D 모델. 물리치료사·동작전문가·애니메이터 제작. 300+ 대학 사용.
- **우리 앱 적용점**: ①"주동근/협력근 구분 + **흔한 실수 + 대체 운동**" 정보 구조는 초보+무릎 보호에 최적 → 운동마다 "이렇게 하면 무릎 위험" 경고와 무릎 친화 대체 운동을 묶어 제시. ②3D 자체 구현은 비현실적이지만 **정보 모델(무엇을 가르칠지)**은 그대로 차용.
- **강점**: 교육적 깊이 최고, 근육 활성 시각화가 부상 예방에 직접 기여.
- **약점/주의**: 3D 자산 제작/라이선스 비용·난이도 매우 높음. 로깅·점진적 과부하 같은 트래커 기능은 약함(학습 도구 성격).
- **수익모델**: 구독(무료 티어 일부 + 인앱 구독).

## 6. Caliber — 비디오 데모 + 코치 폼 리뷰(영상 업로드 피드백)

- **카테고리/플랫폼**: 사이언스 기반 코칭 / iOS·Android·웹
- **핵심기능**: 500+ 운동 **데모 비디오 + 텍스트 지시 + 주동/협력근 표시 + 개인 운동 이력**. 차별점은 **사용자가 자기 폼 영상을 앱에서 촬영·업로드 → 코치가 피드백**(프리미엄/그룹 코칭).
- **우리 앱 적용점**: 인간 코치는 없지만, **"사용자 폼 영상 셀프 기록"** 개념은 가져올 수 있음 — 앱에서 자기 영상을 찍어 데모와 나란히 비교(좌우 분할)하게 하면 초보 폼 자가 점검에 유용. 운동마다 "이력+주동근" 묶음 표시도 참고.
- **강점**: 폼 피드백 루프가 초보 부상 예방에 강력, 무료 트래커 범위도 있음.
- **약점/주의**: 핵심 가치가 사람 코치라 자동화 웹앱으론 1:1 부분 재현 불가.
- **수익모델**: 무료 + 프리미엄(코치) 구독, Pro 그룹 코칭.

## 7. 표현 방식별 장단점 — GIF vs 비디오 vs 3D vs 임베드

| 방식 | 학습 효과 | 구현 난이도 | 저작권/비용 | 단일 웹앱 적합성 |
|---|---|---|---|---|
| **GIF/짧은 루프** | 동작 흐름 직관적, 자동재생·무음 | 낮음 (img 태그) | **자산 출처가 관건** | ★★★★ (오픈 데이터셋 쓰면 최상) |
| **비디오(자체호스팅)** | 디테일·다각도 최고 | 높음(인코딩/CDN/용량) | 촬영·저작권 부담 큼 | ★★ |
| **YouTube 임베드/링크** | 풀 강의·음성 큐 | 낮음(iframe) | 임베드는 비교적 안전, 단 정책 준수 | ★★★★ |
| **3D 모델** | 근육·각도 탐색 최강 | **매우 높음** | 자산 라이선스 고가 | ★ (비현실적) |
| **SVG 근육맵** | "어디에 듣나" 시각화 | 낮음~중 | **오픈소스 다수(자체 제작 가능)** | ★★★★★ |

### YouTube 임베드/링크 — 법적·정책 포인트 (확인됨)
- **임베드 자체**는 "서버 테스트"상 원본이 YouTube 서버에 남으므로 일반적으로 저작권 침해로 보지 않음(하이퍼링크/포인터로 취급). 반면 **영상을 받아 재호스팅하면 위험 급증.**
- 단 **SDNY(뉴욕 남부지법) 등 일부 관할은 서버 테스트를 거부** — 임베드도 공개전시로 본 판례 존재. 관할별 불확실성 있음.
- YouTube **API 서비스 약관/개발자 정책 준수** 필수(커뮤니티 가이드·제3자 권리 침해 금지). 임베드 영상에 YouTube 한정 라이선스 음악/클립이 있으면 유료 서비스 통합 시 추가 권리 이슈 가능.
- **우리 결론**: 한국어·비상업 포털이므로 **공식 iframe 임베드 또는 "유튜브 검색 딥링크"**(`youtube.com/results?search_query=바벨+벤치프레스+자세`)가 가장 안전·저비용. 다운로드·재호스팅은 금지.

## 8. 단일 웹앱에서 "자산 없이" 시각자료 주는 현실적 방법 (핵심)

### A. 데이터/GIF 소스 옵션 (라이선스 확인됨)
- **free-exercise-db (yuhonas)** — **Unlicense(퍼블릭 도메인)**, 800+ 운동 JSON. 필드: `name, force, level, mechanic, equipment, primaryMuscles, secondaryMuscles, instructions, category, images`. 이미지는 GitHub raw URL로 바로 호출(정지 사진 2장/운동). **→ 비상업·자체호스팅에 가장 깨끗한 선택.** (주의: 이미지 개별 라이선스는 명시 불완전하나 데이터셋은 PD)
- **wger** — **AGPL-3.0**, REST API, 자체호스팅 가능. 운동 위키 기여형. (AGPL 전파 조건 주의)
- **ExerciseDB** — 11,000+ 운동 + **GIF**. 라이선스상 "**내 앱 안에서 GIF 표시·상업 사용 가능, 단 원본 파일/DB 재배포·재판매 금지**". RapidAPI 무료 티어는 비상업용. → GIF 품질은 좋으나 **재배포 금지 조항** 때문에 오픈 포털엔 부적합, "앱 내 표시" 용도로만.
- **Kaggle "Fitness Exercises Dataset"**도 동일 ExerciseDB 계열.

### B. 근육 타겟 인체도 (오픈소스, 자체 제작 가능 — 확인됨)
- **react-body-highlighter** (giavinh79) — 근육별 하이라이트, 의존성 최소.
- **react-muscle-highlighter** (soroojshehryar) — 앞/뒤 뷰, 클릭형 근육 영역, 색·강도(intensity) 커스텀, TS·접근성·반응형, **React 외 의존성 0**.
- **body-highlighter** — 프레임워크 무관, 런타임 의존성 0, DOM 직접 렌더.
- **→ 우리의 ⑥시각자료·Fitbod식 피로 히트맵을 자산 비용 0으로 구현 가능.** primaryMuscles/secondaryMuscles 필드를 이 컴포넌트 색강도에 매핑.

### C. 권장 하이브리드 구조 (우리 앱 설계 제안)
운동 상세 카드 = **①SVG 근육맵(주동근 진하게/협력근 연하게)** + **②퍼블릭도메인 정지 이미지(시작/끝 자세 2컷)** + **③단계별 텍스트 큐(한국어, 무릎 보호 경고 포함)** + **④"영상 보기" 버튼 → YouTube 임베드 또는 검색 딥링크**.
- 이렇게 하면 **자체 촬영·3D 자산 0, 저작권 안전, 구현 난이도 낮음**으로 "가르치기"의 4요소(어디에 듣나/어떤 자세/단계 큐/실제 영상)를 모두 충족.
- 무릎 보호 타겟 반영: 운동마다 `kneeRisk` 플래그 + 무릎 친화 **대체 운동**(Muscle & Motion식 구조)을 데이터에 추가.

## 이 카테고리 핵심 교훈 3가지

1. **"한 화면 4종 세트"가 초보 학습의 정답**: 근육맵(어디) + 자세 이미지(무엇) + 텍스트 큐(어떻게) + 영상(실제). Hevy/MuscleWiki/Fitbod 모두 이 조합으로 수렴 — 단, 우리는 자산 없이 SVG+퍼블릭도메인+유튜브로 재현한다.

2. **시각자산은 "만들지 말고 안전하게 빌려라"**: 3D·자체 비디오·GIF 복제는 비용·저작권 지뢰밭. **free-exercise-db(퍼블릭도메인) + 오픈소스 react-body-highlighter + YouTube 공식 임베드**가 비상업 단일 웹앱에 가장 현실적이고 합법적인 스택이다. (ExerciseDB GIF는 재배포 금지라 오픈 포털엔 부적합)

3. **차별점은 "초보·무릎 코칭의 깊이"**: 메이저 앱들의 공통 약점이 "데모는 있지만 폼/안전 코칭이 얕다"는 것. Muscle & Motion식 "주동/협력근 + 흔한 실수 + 대체 운동 + 무릎 위험 경고"를 한국어 텍스트 큐로 채워 넣으면, 자산 부족을 정보 품질로 역전할 수 있다.
